# Admin 대시보드 개발 문서

이 문서는 현재 admin 대시보드를 정리하기 위한 개발 지시서다. 이번 개발 범위는 admin 전체 재설계가 아니라 `admin/dashboard.html` 중심의 대시보드 정리다.

고정 규칙은 `rules.md`를 따른다.

## 1. 개발 목표

admin 대시보드는 복잡한 통계 화면이 아니라 관리자가 바로 확인해야 하는 핵심 업무만 보여주는 화면으로 만든다.

이번 대시보드는 다음 세 섹션으로만 구성한다.

```text
1. 전체 통계
2. 가입 승인
3. 예산 승인
```

지출 검토, 서류 검토, 기업 상세 관리는 이번 대시보드 개발 범위에 넣지 않는다. 해당 기능은 별도 메뉴 또는 후속 개발에서 다룬다.

## 2. 화면 구성

### 2.1 전체 통계 섹션

화면 상단에 배치한다.

목적:

- 서비스의 기본 규모만 빠르게 확인한다.
- 대시보드가 과도한 통계판처럼 보이지 않도록 최소 항목만 둔다.

표시 카드:

- 전체 기업 수
- 진행 중인 사업 수
- 가입 승인 완료 기업 수
- 예산 승인 완료 기업 수

각 카드의 의미:

- 전체 기업 수: 등록된 전체 기업 수
- 진행 중인 사업 수: active 상태인 참가 사업 수
- 가입 승인 완료 기업 수: `approval_status === "approved"`인 기업 수
- 예산 승인 완료 기업 수: `budget_status`가 승인 완료 상태인 기업 수

예산 승인 완료 상태:

```text
budget_approved
change_approved
```

주의:

- 예산 총액, 승인 금액, 잔여 예산 같은 금액 통계는 이번 대시보드에서 제외한다.
- 금액 정보는 기업 목록 또는 기업 상세에서 확인하게 한다.

### 2.2 가입 승인 섹션

전체 통계 아래에 배치한다.

목적:

- 가입 승인 대기 기업을 확인하고 처리한다.

표시 대상:

```text
approval_status === "pending"
```

표시 항목:

```text
기업명 | 대표자 | 참가 사업 | 신청일 | 상태 | 처리
```

처리 기능:

- 가입 승인
- 반려
- 상세 보기

1차 개발 범위:

- `가입 승인` 버튼은 바로 처리 가능하게 한다.
- `반려`는 버튼만 두거나, 가능하면 사유 입력 후 처리한다.
- `상세 보기`는 기업 상세 페이지로 이동한다.

빈 상태 문구:

```text
가입 승인 대기 중인 기업이 없습니다.
```

주의:

- 이 섹션에서는 예산 상태나 지출 상태를 자세히 보여주지 않는다.
- 가입 승인 업무에만 집중한다.

### 2.3 예산 승인 섹션

가입 승인 섹션 아래에 배치한다.

목적:

- founder가 제출한 최초 예산안 또는 예산 변경 요청을 확인한다.
- 대시보드에서는 목록과 진입 동선만 제공하고, 상세 검토는 기업 상세 또는 예산 검토 화면에서 처리한다.

표시 대상:

```text
budget_status === "budget_submitted"
budget_status === "change_submitted"
```

표시 항목:

```text
기업명 | 참가 사업 | 요청 유형 | 제출일 | 요청 사유 | 상태 | 처리
```

요청 유형:

- 최초 예산안
- 예산 변경

요청 유형 판정:

```text
budget_submitted -> 최초 예산안
change_submitted -> 예산 변경
```

처리 기능:

- 검토하기

1차 개발 범위:

- 대시보드에서 바로 승인/보완/반려 버튼을 두지 않는다.
- 예산은 금액 비교가 필요하므로 `검토하기` 버튼으로 기업 상세 또는 예산 검토 화면에 이동한다.

빈 상태 문구:

```text
검토 대기 중인 예산 요청이 없습니다.
```

주의:

- 예산 승인 섹션에는 지출 신청 목록을 섞지 않는다.
- 예산 검토 대기 건만 보여준다.

## 3. 대시보드에서 제외할 것

이번 대시보드에서는 아래 항목을 제외한다.

- 지출 신청 검토 목록
- 첨부서류 검토 목록
- 최근 검토 이력
- 총 승인 예산
- 총 지출 승인액
- 잔여 예산
- 위험 경고 통계
- AI 분석 결과

이 항목들은 후속 admin 기능 또는 기업 상세/지출 검토 메뉴에서 다룬다.

## 4. 데이터 기준

현재 프론트는 mock API 기반이다. 우선 `getAdminDashboard()`가 반환하는 데이터를 사용한다.

필요한 데이터:

- companies
- supportPrograms
- companyCount

기업 데이터에서 필요한 필드:

- id
- name
- representative_name
- created_at
- approval_status
- budget_status
- support_programs.name
- pendingBudgetSubmission.submitted_at
- pendingBudgetSubmission.reason

사업 데이터에서 필요한 필드:

- id
- name
- active

## 5. 권장 렌더링 구조

`src/pages/adminDashboard.js`는 다음 구조로 정리한다.

```text
renderStatsSection()
renderSignupApprovalSection()
renderBudgetApprovalSection()
bindSignupActions()
bindNavigationActions()
render()
```

HTML에는 다음 data target을 두는 것을 권장한다.

```text
data-stats-section
data-signup-approval-section
data-budget-approval-section
```

또는 기존 HTML 구조를 유지하되, 화면상 섹션 제목과 콘텐츠를 위 3개로만 보이게 정리한다.

## 6. UI 방향

### 6.1 전체 톤

- 업무 시스템답게 조용하고 명확하게 만든다.
- 과도한 장식, 큰 히어로, 마케팅식 카드는 사용하지 않는다.
- 표는 촘촘하지만 읽기 쉽게 구성한다.

### 6.2 섹션 제목

섹션 제목은 다음 그대로 사용한다.

```text
전체 통계
가입 승인
예산 승인
```

### 6.3 버튼

가입 승인 섹션:

- `승인`
- `반려`
- `상세 보기`

예산 승인 섹션:

- `검토하기`

### 6.4 상태 라벨

가입 상태:

```text
pending -> 가입 승인 대기
approved -> 가입 승인 완료
rejected -> 가입 반려
```

예산 상태:

```text
budget_submitted -> 예산안 검토 대기
change_submitted -> 예산 변경 검토 대기
budget_approved -> 예산안 승인 완료
change_approved -> 예산 변경 승인 완료
```

가능하면 기존 `budgetStatus.js`의 `getBudgetStatusLabel()`을 사용한다.

## 7. 사용자 흐름

### 7.1 가입 승인 흐름

1. admin이 대시보드에 접속한다.
2. `가입 승인` 섹션에서 승인 대기 기업을 확인한다.
3. `승인`을 누르면 가입 승인 상태가 approved로 바뀐다.
4. 목록이 갱신된다.
5. 승인된 기업은 가입 승인 섹션에서 사라진다.

### 7.2 예산 승인 흐름

1. admin이 대시보드에 접속한다.
2. `예산 승인` 섹션에서 예산 검토 대기 기업을 확인한다.
3. `검토하기`를 누른다.
4. 기업 상세 또는 예산 검토 화면으로 이동한다.
5. 실제 승인/보완/반려는 상세 화면에서 처리한다.

## 8. 개발 완료 기준

다음 조건을 만족하면 이번 대시보드 개발은 완료로 본다.

- 대시보드가 `전체 통계`, `가입 승인`, `예산 승인` 세 섹션으로만 보인다.
- 전체 통계 카드가 4개 표시된다.
- 가입 승인 대기 기업만 가입 승인 섹션에 표시된다.
- 가입 승인 버튼으로 승인 처리가 가능하다.
- 예산 검토 대기 기업만 예산 승인 섹션에 표시된다.
- 예산 승인 섹션의 `검토하기` 버튼이 기업 상세 또는 예산 검토 화면으로 이동한다.
- 지출 검토/서류 검토/금액 통계는 대시보드에서 보이지 않는다.
- 빈 목록일 때 적절한 빈 상태 문구가 표시된다.

## 9. 후속 개발 메모

이번 대시보드 정리 이후 다음 기능을 별도 개발한다.

- 지출 검토 전용 메뉴
- 기업 상세 화면 재정리
- 사업계획서 최종본 admin 관리
- 필수서류 검토 및 다운로드
- 예산 변경 승인 상세 비교
- 검토 이력 및 audit log 정리

이번 작업은 내일 부서와 화면 흐름을 논의하기 위한 최소 admin 대시보드 프로토타입이다.
