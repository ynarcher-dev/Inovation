# Update Log

이 문서는 앞으로 개발하면서 추가할 기능, 변경 요청, 진행 상황을 누적하는 문서다.

변경 가능성이 있는 계획과 요구사항은 이 파일에 기록한다. 절대 바뀌면 안 되는 원칙은 `rules.md`에 기록하고, 일반 업데이트 과정에서 수정하지 않는다.

## 다음 개발 우선순위

### [x] 코드 중복 제거 및 오류 처리 정리

- R2 presigned upload/delete 서명 로직을 Supabase Edge Function 공용 모듈로 통합했다.
- 비동기 버튼/모달 처리 중 발생하는 오류를 화면에 표시하도록 정리했다.
- 사업계획서 파일명 표시 필드를 실제 DB 컬럼인 `original_filename`으로 수정했다.
- 관리자 기업 상세의 보완/반려 히스토리를 실제 `reviews` 데이터와 연결했다.

### [x] 창업자 프로필 관리

- 창업자 사이드바의 `새 지출 신청` 링크를 제거하고 `프로필` 링크로 교체했다.
- 창업자가 기업명, 대표자명, 사업자등록번호, 연락처를 수정할 수 있는 프로필 화면을 추가했다.
- 창업자 본인 기업만 수정할 수 있는 Supabase RPC를 추가했다.

### [x] 공통 규정 및 유의사항 관리

- 관리자가 별도 관리 화면에서 규정 및 유의사항을 추가/삭제할 수 있게 했다.
- 등록된 항목은 모든 창업자 대시보드의 `규정 및 유의사항` 영역에 표시된다.
- 기존 고정 문서 링크는 DB 초기 데이터로 이전했다.
- 관리자 왼쪽 사이드바에 `규정 및 유의사항` 메뉴를 추가하고 별도 관리 화면으로 분리했다.

### [x] 참가 사업명 관리

- 창업자 가입 화면에서 참가 사업명을 select box로 선택하게 했다.
- 관리자가 왼쪽 사이드바의 `참가 사업명 관리` 화면에서 참가 사업명을 추가하거나 비활성화할 수 있게 했다.
- 초기 참가 사업명으로 `체육인 창업지원`, `예술인 창업지원`, `제주도민 창업지원`을 DB에 추가했다.
- 참가 사업 코드는 DB에서 자동 생성하고, 관리자 화면에서는 읽기 전용으로 표시한다.
- 참가 사업명 삭제 시 기존 데이터 영향 가능성을 강하게 경고한다.
- 사업코드를 사업명 입력창 아래에 배치하고, 삭제 시 관리자 비밀번호 확인을 요구한다.
- 이메일 확인이 켜진 가입에서도 Auth 사용자 생성 직후 기업/프로필/멤버십이 DB에 저장되도록 가입 트리거를 추가한다.

### [x] 관리자 가입 신청 관리

- 관리자 왼쪽 사이드바에 `가입 신청 관리` 메뉴를 추가했다.
- 가입 신청 관리 화면에서 승인 대기 기업을 승인하거나 반려할 수 있게 했다.
- 전체 가입 기업의 승인 상태와 참가 사업명을 확인할 수 있게 했다.

### [ ] 파일 업로드 운영 고도화

- 업로드된 파일의 다운로드 URL 발급 흐름을 추가한다.
- 업로드된 파일 삭제 UI와 `delete-uploaded-file` Edge Function 호출을 연결한다.
- R2 bucket CORS와 Supabase Edge Function 환경변수 설정이 운영/로컬 환경에서 정상 동작하는지 검증한다.
- 관리자 화면에서 업로드된 파일 목록과 원본 확인 액션을 제공한다.

### [ ] 관리자 검토 고도화

- 관리자 검토 화면에서 누락 서류, 위험 경고, 사업계획서 항목 적합성을 한 화면에 보여준다.
- 보완 요청 메시지를 창업자 대시보드와 상세 화면에 즉시 노출한다.
- 상태 변경을 `audit_logs`에 기록한다.
- 기업 가입 반려 기능을 추가하고 승인/반려 이력을 감사 로그와 관리자 메모로 확장한다.

### [ ] 사업계획서 관리

- 관리자만 기업별 최종 사업계획서를 업로드/교체할 수 있게 한다.
- 사업계획서 변경 이력을 남긴다.
- 사업계획서 항목별 비목과 배정 금액을 관리자 화면에서 생성, 수정, 삭제할 수 있게 한다.
- 사업계획서 원본 파일 다운로드 또는 미리보기 기능을 추가한다.

### [ ] Gemini AI 검토

- AI는 최종 승인자가 아니라 검토 보조자로만 사용한다.
- 업로드 파일의 문서 유형, 업체명, 금액, 날짜, 날인 여부를 추출한다.
- 누락 서류와 금액 불일치 가능성을 표시한다.
- AI 결과에는 신뢰도와 원본 파일 확인 안내를 함께 표시한다.

### [ ] 집행/검수/정산 단계

- 사전승인 완료 후 집행 증빙 제출 단계를 추가한다.
- 검수조서 작성/업로드 흐름을 만든다.
- 정산 내역서 작성 또는 엑셀 업로드 기능을 만든다.
- 최종 제출 패키지 다운로드 기능을 추가한다.
